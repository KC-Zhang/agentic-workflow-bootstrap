# Patterns

Use this directory for reusable implementation patterns that are too specific for generic code style docs.

Add a pattern when:

- The same approach is used across multiple files or modules.
- A repeated bug or review comment needs durable guidance.
- Future agents need examples for a fragile integration.
- The pattern affects data access, API design, async workflows, testing, security, privacy, cost, or deployment.

Do not add a pattern for one-off implementation details. Keep local facts in module docs.

Recommended structure:

```markdown
# Pattern Name

## Use When

## Do

## Avoid

## Example

## Related Docs
```
