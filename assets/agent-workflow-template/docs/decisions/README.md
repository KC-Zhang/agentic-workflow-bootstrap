# Architecture Decisions

Use this directory for long-term architecture decisions.

Create `docs/decisions/YYYY-MM-DD-title.md` when:

- A core dependency, platform, provider, database, deployment model, auth system, payment system, or cross-module contract changes.
- Multiple reasonable options exist and future maintainers need to know why one was chosen.
- The decision creates lasting constraints or migration obligations.

Recommended structure:

```markdown
# YYYY-MM-DD - Decision Title

## Status

accepted / superseded / deprecated

## Context

## Decision

## Alternatives Considered

## Consequences

## Related Docs
```
